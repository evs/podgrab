   ```go

   // Every N minutes: checkFrequency
   freqExpr := fmt.Sprintf("*/%d * * * *", checkFrequency)
   c.AddFunc(freqExpr, service.RefreshEpisodes)
   c.AddFunc(freqExpr, service.DownloadMissingEpisodes)
   c.AddFunc(freqExpr, service.CheckMissingFiles)
   c.AddFunc(freqExpr, service.DownloadMissingImages)

   doubleFreqExpr := fmt.Sprintf("*/%d * * * *", checkFrequency*2)
   c.AddFunc(doubleFreqExpr, service.UnlockMissedJobs)

   tripleFreqExpr := fmt.Sprintf("*/%d * * * *", checkFrequency*3)

   c.AddFunc("0 0 */2 * *", service.CreateBackup)

The main goroutine should continue to start the Gin server.

Replace gorilla/websocket with coder/websocket (nhooyr.io/websocket) per D-07. The coder/websocket API differs significantly from gorilla/websocket, so the full file must be adapted.

Key API differences:
- `websocket.Upgrader` → `websocket.Accept(w, r, &websocket.AcceptOptions{})`
- `conn.ReadJSON(&msg)` → use `websocket.MessageType` and `json.NewDecoder(conn).Decode(&msg)` inside a read loop
- `conn.WriteJSON(msg)` → `json.NewEncoder(conn).Encode(msg)` with `websocket.MessageText` write
- `conn.Close()` → `conn.Close(websocket.StatusNormalClosure, "")`
- Connection upgrade: gorilla uses `upgrader.Upgrade(w, r, nil)`, coder uses `websocket.Accept(w, r, opts)`
- Message types: coder/websocket uses `websocket.MessageText` and `websocket.MessageBinary` instead of `gorilla.TextMessage`

   **Imports:**
   - Remove `"github.com/gorilla/websocket"`
   - Add `"context"`, `"nhooyr.io/websocket"`, `"nhooyr.io/websocket/wsjson"`

   **Type changes:**
   - Replace `*websocket.Conn` with `*websocket.Conn` from `nhooyr.io/websocket` throughout. The types are different — `nhooyr.io/websocket.Conn` is not interface-compatible with `gorilla/websocket.Conn`
   - The `Message` struct field `Connection *websocket.Conn` stays but now uses nhooyr's Conn type

   **Connection upgrade (Wshandler function):**
   ```go
   func Wshandler(w http.ResponseWriter, r *http.Request) {
       conn, err := websocket.Accept(w, r, &websocket.AcceptOptions{
           InsecureSkipVerify: true, // matches current gorilla behavior (no origin check)
       })
       if err != nil {
           fmt.Println("Failed to set websocket upgrade:", err)
           return
       }
       defer conn.Close(websocket.StatusNormalClosure, "")
       
       ctx := context.Background()
       for {
           var mess Message
           err := wsjson.Read(ctx, conn, &mess)
           if err != nil {
               isPlayer := activePlayers[conn] != ""
               if isPlayer {
                   delete(activePlayers, conn)
                   broadcast <- Message{
                       MessageType: "PlayerRemoved",
                       Identifier:  mess.Identifier,
                   }
               }
               delete(allConnections, conn)
               break
           }
           mess.Connection = conn
           allConnections[conn] = mess.Identifier
           broadcast <- mess
       }
   }
   ```

   **HandleWebsocketMessages function:**
   - Replace `connection.WriteJSON(msg)` with `wsjson.Write(ctx, connection, msg)` for each WriteJSON call
   - Add `ctx := context.Background()` at the beginning of the function
   - Example for PlayerExists broadcast:

   case "RegisterPlayer":

           wsjson.Write(ctx, connection, Message{
               Identifier:  msg.Identifier,
               MessageType: "PlayerExists",
           })
       }
   ```

   **Important:** The `wsjson.Write` and `wsjson.Read` functions use context for cancellation. Use `context.Background()` for now. Phase 3 will add proper context handling with request-scoped timeouts.
