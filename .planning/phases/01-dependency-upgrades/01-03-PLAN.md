<interfaces>
From RESEARCH.md (GORM v2 critical changes):
- BlockGlobalUpdate is now default — Delete/Update without WHERE silently does nothing
- Model tags changed from snake_case to camelCase — auto_increment, unique_index silently ignored
- db.CreateTable() / db.DropTable() no longer exist → use db.Migrator()
- TableName() return value now cached — dynamic table names break
- Soft Delete requires gorm.DeletedAt type, not just a field named DeletedAt
- Hook signatures changed from func(u *User) error to func(tx *gorm.DB) error
- Count() only accepts *int64 — existing int or int32 calls fail silently
- Method chain safety: reusing *gorm.DB after .Where() in goroutines causes data races

Current GORM model patterns (db/base.go):
```go
type Base struct {
    ID        string `sql:"type:uuid;primary_key"`
    CreatedAt time.Time
    UpdatedAt time.Time
    DeletedAt *time.Time `gorm:"index"`
}
```

Current GORM init (db/db.go — pattern):
```go
db, err := gorm.Open(sqlite.Open(dbPath), &gorm.Config{})
```

3. Run `go build ./...` — it WILL have compilation errors. Fix them in the following steps.

**Step B: Fix GORM v2 hook signatures**
The `BeforeCreate` hook in `db/base.go` already uses the v2 signature `func (base *Base) BeforeCreate(tx *gorm.DB) error` (it was written for v1 with the v2-compatible signature). Verify this is correct. If any other hooks exist with v1 signatures, update them.

**Step C: Fix GORM model struct tags**
GORM v2 changed struct tag format. Search ALL model structs for `snake_case` GORM tags that need to change:

Old v1 tags → New v2 equivalents:
- `sql:"type:uuid;primary_key"` → `gorm:"type:uuid;primaryKey"` (in Base struct)
- `gorm:"index"` → `gorm:"index"` (this one is FINE, didn't change)
- Any `auto_increment` → `autoIncrement`
- Any `unique_index` → `uniqueIndex`
- Any `polymorphic_value` → `polymorphicValue`

Read every model file (`db/podcast.go`, `db/podcastItem.go`, `db/settings.go`, `db/tag.go`) and audit all struct tags.

**Step D: Fix GORM v2 API changes in db/dbfunctions.go**

Search for and fix these specific GORM v2 incompatibilities:

1. **`db.CreateTable()` / `db.DropTable()`**: Replace with `db.Migrator().CreateTable(&Model{})` / `db.Migrator().DropTable(&Model{})`

2. **`Count()` with `int`**: Search for any `Count()` calls that pass non-`*int64` variables. In v2, `Count()` ONLY accepts `*int64`. Change `var count int` to `var count int64` and update any `count` variable types.

3. **`Delete()`/`Update()` without `WHERE`**: GORM v2's `BlockGlobalUpdate` mode is default. Find ANY `db.Delete()` or `db.Update()` calls without a `.Where()` clause and either:
   - Add a `.Where()` condition if the intent is specific
   - Add `.AllowGlobalUpdate()` if the intent IS to delete/update all rows
   Search `db/dbfunctions.go` and all controller files for bare `.Delete()` calls.

4. **Method chain safety**: If `db.DB` is reused across goroutines (it is — it's a global singleton), wrap reads in `db.Session(&gorm.Session{})` to prevent chain pollution. For this phase, just add `Session` wrapper where queries are built in handlers that run concurrently. The global singleton issue is addressed in a later phase.

5. **`DeletedAt` field**: Check if `Base.DeletedAt` is `*time.Time` or `gorm.DeletedAt`. In v2, soft delete requires `gorm.DeletedAt` type. Current code has `*time.Time` — change to `gorm.DeletedAt`:
   ```go
   type Base struct {
       ID        string         `gorm:"type:uuid;primaryKey"`
       CreatedAt time.Time
       UpdatedAt time.Time
       DeletedAt gorm.DeletedAt `gorm:"index"`
   }
   ```

**Step E: Fix GORM initialization in db/db.go**

Update the GORM Open call and add SQLite WAL mode:
```go
import (
    "gorm.io/gorm"
    "gorm.io/driver/sqlite"
)

db, err := gorm.Open(sqlite.Open(dbPath), &gorm.Config{})
if err != nil {
    // Fatal exit on DB init failure (per D-13 and BUG-06 from future phases)
    log.Fatalf("failed to connect to database: %v", err)
}

// Enable WAL mode for better concurrent read performance
sqlDB, _ := db.DB()
sqlDB.Exec("PRAGMA journal_mode=WAL")
sqlDB.Exec("PRAGMA busy_timeout=5000")
```
